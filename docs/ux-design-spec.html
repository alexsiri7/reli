Briefing as landing page: The main content area shows the briefing by default (not the chat). The sidebar stays visible on desktop with the type-organized Things list. Each briefing item has inline actions — Done, Snooze, Dismiss, Chat about it. Mobile gets its own dedicated Briefing tab (3rd tab added to the bottom bar).

Briefing design rules

  • The greeting adapts: "Good morning" / "Good afternoon" / "Good evening" based on time
  • Cards are ordered by urgency: schedule → due items → needs attention → learned preferences
  • The "I Noticed" card only appears when there's a new insight — not every day
  • Proactive nudge banner appears at top only when there's a time-sensitive suggestion
  • "Stop these" on nudges feeds directly into preference learning (negative signal)
  • Clicking "Chat about it" switches to chat view with the Thing pre-loaded as context
  • The briefing should fit in one screen on desktop, scrollable on mobile

Command palette behavior

  • Cmd+K / Ctrl+K opens the palette from anywhere
  • Fuzzy search across Thing titles, types, and content
  • Results grouped: Things first, then contextual actions, then quick actions
  • Arrow keys to navigate, Enter to select, Esc to close
  • Type > prefix to filter to actions only (like VS Code)
  • Type # prefix to filter by type (#task, #project, #person)
  • Recent items shown when palette is empty (before typing)

Nudge types and their triggers

  • Time-based: Calendar events (30 min before), task check-in dates (morning of)
  • Context-enriched: Attach relevant Things to reminders (last conversation with a person, related notes)
  • Opportunity: Price drops, contract expirations, pattern-matched insights
  • Learning: New preference detected, showing "That's right" / "Not really" feedback

Delivery channels (progressive)

  • Phase 1: In-app nudge banner at top of chat/briefing
  • Phase 2: Browser push notifications (requires user permission)
  • Phase 3: Telegram bot messages, email digest
  • Every nudge type has a "Stop these" option that feeds into preference learning
  • Target: <3 nudges per day to avoid fatigue

Typography Scale

Briefing greeting — 22px / 700
Section heading — 18px / 700
Panel title — 16px / 600
Body text — 14px / 400
Secondary text — 13px / 400
Caption — 12px / 400
Section label — 11px / 600 / uppercase

Spacing

  • 4px — Tight gaps (between badge and text, icon and label)
  • 8px — Standard inner padding (list items, small cards)
  • 12px — Mobile section padding, compact card padding
  • 16px — Desktop section padding, sidebar item padding, standard gaps
  • 20px — Chat message padding, panel content padding
  • 24px — Briefing content padding, large section margins
  • 32px — Major section dividers

Radius

  • 4px — Checkboxes, tiny elements
  • 6px — Small buttons, badges, tags
  • 8px — Input fields, search bars, small cards
  • 12px — Cards, panels, chat bubbles
  • 16px — Command palette, large modals
  • 20px — Chat input container
  • 50% — Avatars, FAB, circular buttons

Responsive Breakpoints

  • < 768px (below md) — Mobile layout: single panel + bottom tab bar (3 tabs: Things, Chat, Briefing)
  • ≥ 768px (md) — Desktop layout: sidebar + main content + optional detail panel
  • ≥ 1024px (lg) — Comfortable three-panel layout (sidebar + chat + detail)
  • Mobile: bottom tab bar replaces sidebar toggle; FAB for quick-add; detail panel as full overlay
  • Desktop: resizable sidebar (200-500px); detail panel docks right; command palette centered
